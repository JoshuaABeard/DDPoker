# Web Client Feature Parity R6 — Design

**Date:** 2026-02-28
**Status:** APPROVED

## Overview

Round 6 of web client feature parity focuses on three features that exist in the desktop Java
client but are missing from the web client:

1. **Statistics Viewer** — Per-player tournament stats embedded in the profile page
2. **Tournament Profiles** — Server-stored reusable game configuration templates
3. **Showdown Analysis** — Opponent hand specification in the equity simulator

## 1. Statistics Viewer (Profile-Embedded)

### Location

New section in `/online/myprofile` page, between Alias Management and Quick Links.

### Summary Stats Bar

Computed client-side from tournament history data (`/api/history`):

- Total Games Played
- Average Finish Position
- Total Profit/Loss (sum of prize minus buy-in)
- Best Finish
- Win Rate (1st place finishes / total games)

### Profit/Loss Trend Chart

- Line chart showing cumulative profit/loss over time
- X-axis: game date, Y-axis: cumulative P/L
- Green line when positive, red when negative
- Custom SVG implementation (no charting library dependency)
- Placed between summary stats bar and history table

### Tournament History Table

Sortable table with columns: Finish, Tournament Name, Date, Buy-in, Prize, Profit/Loss.

- Click column headers to sort
- Pagination (reuses existing `page`/`pageSize` params from `/api/history`)
- Optional date range filter (reuses existing `from`/`to` params)

### Implementation

Pure frontend — new `PlayerStatsSection` component. Calls `tournamentApi.getHistory()` with
current username. No server changes needed.

## 2. Tournament Profiles (Server-Stored)

### Server Side

**New table:**

```sql
CREATE TABLE IF NOT EXISTS wan_template (
    wtp_id         BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    wtp_profile_id BIGINT NOT NULL REFERENCES wan_profile(wpr_id),
    wtp_name       VARCHAR(100) NOT NULL,
    wtp_config     TEXT NOT NULL,
    wtp_created    TIMESTAMP NOT NULL,
    wtp_modified   TIMESTAMP NOT NULL
);
```

**New entity:** `TournamentTemplate` — follows `OnlineProfile` JPA entity pattern with
`@PrePersist`/`@PreUpdate` for timestamps.

**New repository:** `TournamentTemplateRepository extends JpaRepository<TournamentTemplate, Long>`

**New REST endpoints** (authenticated, under `/api/profile/templates`):

| Method | Path | Description |
|--------|------|-------------|
| GET | `/api/profile/templates` | List current user's templates |
| POST | `/api/profile/templates` | Create template (name + config JSON) |
| PUT | `/api/profile/templates/{id}` | Update template |
| DELETE | `/api/profile/templates/{id}` | Delete template |

Config is stored as a JSON blob containing: maxPlayers, startingChips, gameType,
blindStructure, aiConfig, payouts, advanced settings, practice options.

### Client Side

**Create Game page** — new "Templates" toolbar at top of form:

- Dropdown to select a saved template
- "Load" button — populates all form fields from template config
- "Save Current Settings" button — saves current form state (prompts for name)
- "Delete" button — removes selected template (with confirmation)

**Profile page** — new "Saved Templates" subsection showing template count.

### Web API client

New `templateApi` in `lib/api.ts`:

```typescript
export const templateApi = {
  list: () => apiFetch<TemplateDto[]>('/api/profile/templates'),
  create: (name: string, config: object) => apiFetch<TemplateDto>('/api/profile/templates', { method: 'POST', body: { name, config } }),
  update: (id: number, name: string, config: object) => apiFetch<TemplateDto>(`/api/profile/templates/${id}`, { method: 'PUT', body: { name, config } }),
  delete: (id: number) => apiFetch<void>(`/api/profile/templates/${id}`, { method: 'DELETE' }),
}
```

## 3. Showdown Analysis (Simulator Extension)

### Equity Calculator Extension

Update `calculateEquity()` to accept optional known opponent hands:

```typescript
export function calculateEquity(
  holeCards: string[],
  communityCards: string[],
  numOpponents: number,
  iterations: number = 10000,
  knownOpponentHands?: string[][]
): EquityResult
```

Behavior:
- Known opponent hands are removed from deck before shuffling
- Known opponents are dealt their specified cards
- Remaining opponents (if `knownOpponentHands.length < numOpponents`) get random cards
- Return type extended with per-opponent breakdown

```typescript
interface EquityResult {
  win: number
  tie: number
  loss: number
  iterations: number
  opponentResults?: { win: number; tie: number; loss: number }[]
}
```

### Simulator UI Extension

Below the opponent count selector, add expandable "Specify Opponent Hands" section:

- For each opponent (1 to N): two card slots with CardPicker
- Cards already selected by player or other opponents are unavailable
- "Clear" button per opponent to revert to random
- Visual indicator: specified opponents show cards, random opponents show card backs
- Results section shows per-opponent win/tie/loss when hands are specified

## Key Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Stats computation | Client-side | Reuses existing `/api/history` API, no server changes |
| Chart library | Custom SVG | Avoids dependency for a simple line chart |
| Template storage | Server-side (H2) | Persists across devices, requested by user |
| Template config format | JSON blob | Simple, flexible, no schema migration on config changes |
| Showdown simulation | Monte Carlo with known hands | Consistent with existing calculator, simple extension |
| Showdown UI | Extend existing Simulator | Avoids duplicate UI, natural feature home |
